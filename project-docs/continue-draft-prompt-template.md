# Continue Draft Prompt Template

Используйте этот шаблон, когда нужно быстро получить черновик статьи прямо в открытый `.md`-файл.

Замените все блоки в угловых скобках на свои данные.

```text
Нужен черновик статьи для сайта о фотографии.

Тема статьи:
<TITLE>

Краткое описание:
<DESCRIPTION>

Структура и обязательные блоки:
<OUTLINE>

Что важно раскрыть:
<KEY_POINTS>

Ограничения:
- не выдумывай факты
- не добавляй то, чего нет в заданной структуре
- пиши связный черновик на русском языке
- избегай воды, штампов и рекламной риторики
- держи популярную, но содержательную подачу
- обязательно встраивай ненавязчивую, но твёрдую «мотивацию обучения»: читатель должен понимать, зачем осваивать описываемый инструмент, приём или способ видеть кадр
- такая «мотивация обучения» должна работать через практическую ценность темы, а не через рекламные обещания, лозунги или прямые призывы

Технические требования:
- не добавляй front matter
- не добавляй заголовок статьи уровня H1
- начинай сразу с текста статьи
- используй подзаголовки уровня `##` и `###`, если это нужно по структуре
- не предлагай варианты названия статьи
- считай, что заголовок уже выбран и зафиксирован вне этого черновика
- объём текста должен строго соответствовать заданному в промпте
- не сокращай ответ до краткой заметки, тезисного плана или обзорного наброска
- нужен полный развёрнутый черновик статьи, а не заготовка
- каждый смысловой раздел должен быть раскрыт полноценно, а не в 1-2 абзацах-заглушках
- если возникает соблазн ответить короче, всё равно выдай полный текст в указанном диапазоне объёма

Формат ответа:
- выдай результат как чистый Markdown-текст для прямой вставки в открытый `.md`-файл
- не используй тройные кавычки, code block и обрамление ответа
- не добавляй вступление, пояснения перед текстом и комментарии после текста
- не пиши "вот статья", "ниже черновик" и другие служебные фразы
- не добавляй служебные пометки о структуре ответа
- весь ответ должен быть сразу пригоден для команды `Insert at cursor` без дополнительной очистки

Контекст проекта:
<PROJECT_RULES_OR_NOTES>

Объём:
<TARGET_LENGTH>

Это обязательный диапазон, а не ориентир. Если текст заметно короче нижней границы, задача не выполнена.

Сделай полный цельный черновик статьи в указанном диапазоне объёма.
```

## Быстрый пример заполнения

```text
<TITLE> = Как использовать окно как основной источник света
<DESCRIPTION> = Черновик статьи для блога GurFoto
<OUTLINE> =
- Вход: почему окно остаётся рабочим источником света
- Разбор: направление света, расстояние, фон, экспозиция
- Практика: три коротких сценария съёмки
- Ошибки: контраст, фон, неверная точка съёмки
- Вывод
<KEY_POINTS> =
- показать, что окно не равно мягкий свет по умолчанию
- объяснить роль расстояния до окна
- не уходить в студийные схемы
<PROJECT_RULES_OR_NOTES> =
- ориентируйся на ai-rules.md, article-template.md и style-reference.md
- не используй рекламный тон
<TARGET_LENGTH> = 2200-2600 слов
```
